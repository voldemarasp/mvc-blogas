µFrame

µFrame is a minimal PHP MVC framework.
The best tool to learn Model View Controller design pattern or just a template for Your daily projects.

Installation and configuration

Just unzip it to Your shared hosting directory.
That's it. No Composer, no CLI, no distraction.
Should You need database (PHP PDO) functionality, update config.php file with Your database credentials.

MVC Design

Models

Models are the classes responsible for data exchange.
Most common use is to send and receive data from database(s).

Views

Views are the HTML5 templates.

Controllers

Controllers are responsible for most of the logic and data manipulation. It chooses what view to use (if at all) and where to get the data.

Assets

All CSS, JavaScript and image files resides in /assets/ subdirectories and are included in views (templates).

Routing

If user comes to https://example.com/Products/showAll/10 address, he weill be pointed to Products controller and showAll() method.
Everything after https://example.com/Products/showAll/ will be split on / character and passed as argument sequence to showAll() method.

Core Classes

Database

Database Core class has four methods: select(), insert(), update() and remove().
$db object can be used in any model in order to communicate with database.
It was inteded to use with prepared statements, where query parameters are detached from SQL query itself like this:
$db->select("SELECT * FROM blog_posts WHERE user_id = :user_id", ["user_id" => 123]);
As You can see we are using :user_id placeholder in the query and 123 is transfered in the array as a parameter.

Log

µFrame has built in basic loging suport.
Should You like to put something in the log.txt file, just call $log->error(), $log->warning() or $log->debug() and pass string as a parameter.
